# 📢 TideLog 内测沟通话术

---

## 核心策略

**定位**：你不是在「推销」，而是在「邀请共创」。内测用户是产品的早期参与者，不是单纯的免费使用者。

**核心逻辑**：
1. 先筛选 → 只找真正会用的人
2. 坦诚告知 → 将来要付费，现在免费
3. 互惠承诺 → 激活码换反馈，双方有价值

---

## 场景一：回复评论区留言

> 适用于：有人在你的帖子下面留言「想用」「求链接」「怎么下载」

### 模板 A：筛选 + 邀请

```
感谢关注！TideLog 目前还在内测阶段 🧪

想先确认一下：
1. 你现在在用 Obsidian 吗？
2. 平时有写日记/做每日计划的习惯吗？

如果都是的话，私信我，我可以给你发一个免费的内测激活码 🎁
```

> 💡 这条回复的目的：通过两个问题筛出真正的目标用户（已有 Obsidian + 有记录习惯），同时避免直接给链接导致收集不到反馈。

### 模板 B：简洁版

```
正在内测中！私信我你的 Obsidian 使用情况，符合条件可以拿到免费激活码 ✨
```

## 场景二：私信沟通（核心对话）

> 适用于：通过评论区筛选后进入私信，或者直接私信来的人

### 第 1 步：了解对方

```
Hi～ 感谢对 TideLog 的关注！

在发安装包之前，我想先了解一下：
1. 你用 Obsidian 多久了？
2. 你现在有在用哪些插件做日记或计划管理？
3. 你平时是在电脑上用还是手机上也用？

这样我可以帮你更好地上手 😊
```

> 💡 目的：了解用户经验水平，判断是否是目标用户，同时收集竞品信息。

### 第 2 步：说明付费计划 + 发放激活码

```
了解了！看起来很适合试试 TideLog 👍

跟你先说明一下：
TideLog 之后正式上架会是付费插件（年费制），但目前内测阶段，我会给你一个 **免费的一年激活码**。

我唯一的希望是：在这一年里你能偶尔给我一些使用反馈 —— 遇到 bug、觉得哪里不好用、或者有什么想法都可以随时告诉我。不用写得很正式，语音/截图/几句话都行。

这对我来说非常有价值，你的反馈会直接影响到产品的方向 💪

如果你觉得 OK，我这就把安装包和激活码发给你！
```

> 💡 目的：
> - 提前管理预期（未来要收费）
> - 让用户感受到「被信任」和「有参与感」
> - 用「我唯一的希望」降低反馈的心理压力
> - 不要求太多，只要「偶尔反馈」

### 第 3 步：发送安装包和激活码

```
这是安装指南和下载地址 📦

[贴上安装指南链接或截图]

你的专属激活码：XXXX-XXXX-XXXX-XXXX

遇到任何问题直接找我，我随时在 👋

另外我做了一个反馈模板，你可以参考（也可以不按这个格式，怎么方便怎么来）：

[贴上反馈模板链接或截图]
```

## 场景三：对方犹豫 / 追问价格

> 适用于：有人想了解但担心付费的情况

```

现在就是最好的来免费体验的时候 ✨
```

## 场景四：对方明显不是目标用户

> 适用于：没有用过 Obsidian / 只是随便问问

```
感谢关注！TideLog 需要配合 Obsidian 使用，如果你还没有用过 Obsidian，可以先去 obsidian.md 了解一下 😊

后续正式上架的时候我会再发帖通知～
```

> 💡 目的：礼貌但明确地筛掉非目标用户，避免浪费双方时间。

## 场景五：用户反馈后的回复

> 建立长期关系，让人愿意持续反馈

### 收到 Bug 反馈

```
感谢反馈！这个 bug 我记下了，会在下个版本修复 🛠️
修好了第一时间通知你～
```

### 收到功能建议

```
这个想法很好！我记下来了 📝
你是第 N 个提这个需求的人，看来确实很需要，我会认真考虑加进去 💡
```

### 收到好评

```
太开心了！知道有人在用而且觉得好用，是我最大的动力 😄
继续用，有任何想法随时告诉我～
```

## 沟通注意事项

1. **语气自然真诚**：不要太正式、不要像客服话术，像朋友聊天一样
2. **回复要及时**：内测用户的消息最好当天回复，建立信任
3. **记录每个用户**：建议用文档记录每位测试者的设备、问题、反馈，方便追踪
4. **适度跟进**：发出激活码后 3-5 天可以主动问一下「安装成功了吗？上手顺利吗？」
5. **尊重边界**：如果对方不想反馈也不要追着问，好的体验自然会让人愿意说
